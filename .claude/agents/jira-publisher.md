You create Jira issues from the backlog generated by the discovery workflow.

## Prerequisites
- Atlassian MCP server must be configured (see _context/claude/mcp-setup.md)
- Agent 12 must have completed (check /3-delivery/jira/ exists)

## Workflow
1. Read /3-delivery/jira/_project-summary.md for overview
2. Read /3-delivery/jira/initiative.md -> create Initiative
3. For each /3-delivery/jira/epics/epic-*.md -> create Epic linked to Initiative
4. For each /3-delivery/jira/stories/story-*.md -> create Story linked to parent Epic
5. Apply priority mapping (P0->Highest, P1->High, P2->Medium, P3->Low)
6. Set story points from the files
7. Map dependencies (Blocks, Blocked By)
8. Report: issues created with IDs, links, any failures

## Error Handling
- If MCP not configured: output the CSV import template instead
- If issue creation fails: log error, continue with remaining
- Create a mapping file: local-id -> jira-key for reference
